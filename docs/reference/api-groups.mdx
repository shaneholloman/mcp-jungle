---
title: "REST API: tool group management endpoints"
description: "Create, list, update, and delete tool groups. Resolve effective tools per group and connect MCP clients to group-scoped MCP proxy endpoints."
---

Tool groups let you expose a curated subset of tools to specific MCP clients. Each group has its own MCP proxy endpoint, so an AI agent pointed at a group only sees the tools you chose. Groups are additive — you include tools by name or by server, then optionally exclude individual tools from server-level inclusions.

All group management endpoints require **admin** role.

## `GET /api/v0/tool-groups`

Lists all tool groups with their inclusion and exclusion configuration.

**Access:** admin only

```bash cURL
curl http://localhost:8080/api/v0/tool-groups \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Response `200`**

```json
[
  {
    "name": "read-only",
    "description": "Read-only filesystem tools",
    "included_tools": ["filesystem__read_file", "filesystem__list_directory"],
    "included_servers": [],
    "excluded_tools": []
  },
  {
    "name": "all-weather",
    "description": "Everything from the weather server",
    "included_tools": [],
    "included_servers": ["weather"],
    "excluded_tools": ["weather__get_raw_data"]
  }
]
```

<ResponseField name="name" type="string" required>
  Unique name of the tool group.
</ResponseField>

<ResponseField name="description" type="string">
  Human-readable description.
</ResponseField>

<ResponseField name="included_tools" type="string[]">
  Explicitly included tool names.
</ResponseField>

<ResponseField name="included_servers" type="string[]">
  Server names whose tools are included wholesale.
</ResponseField>

<ResponseField name="excluded_tools" type="string[]">
  Tool names excluded from the group, applied after all inclusions. Useful for removing specific tools when an entire server is included.
</ResponseField>

## `POST /api/v0/tool-groups`

Creates a new tool group.

**Access:** admin only

### Request body

<ParamField body="name" type="string" required>
  Unique name for the group.
</ParamField>

<ParamField body="description" type="string">
  Optional human-readable description.
</ParamField>

<ParamField body="included_tools" type="string[]">
  List of fully qualified tool names to include, e.g. `["filesystem__read_file"]`.
</ParamField>

<ParamField body="included_servers" type="string[]">
  List of server names. All currently enabled tools from each server are included.
</ParamField>

<ParamField body="excluded_tools" type="string[]">
  Tool names to exclude. Exclusions are applied after all inclusions, so a tool listed here will not appear even if its server is in `included_servers`.
</ParamField>

<Tabs>
  <Tab title="By tool names">

```bash cURL
curl -X POST http://localhost:8080/api/v0/tool-groups \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "read-only",
    "description": "Read-only filesystem tools",
    "included_tools": [
      "filesystem__read_file",
      "filesystem__list_directory"
    ]
  }'
```

  </Tab>
  <Tab title="By server (with exclusion)">

```bash cURL
curl -X POST http://localhost:8080/api/v0/tool-groups \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "all-weather",
    "description": "Everything from the weather server",
    "included_servers": ["weather"],
    "excluded_tools": ["weather__get_raw_data"]
  }'
```

  </Tab>
</Tabs>

**Response `201`**

<ResponseField name="streamable_http_endpoint" type="string" required>
  URL of the streamable HTTP MCP proxy for this group. Point your MCP client here.
</ResponseField>

<ResponseField name="sse_endpoint" type="string" required>
  URL of the SSE MCP proxy for this group (legacy transport).
</ResponseField>

<ResponseField name="sse_message_endpoint" type="string" required>
  SSE message handler URL (companion to `sse_endpoint`).
</ResponseField>

```json
{
  "streamable_http_endpoint": "http://localhost:8080/v0/groups/read-only/mcp",
  "sse_endpoint": "http://localhost:8080/v0/groups/read-only/sse",
  "sse_message_endpoint": "http://localhost:8080/v0/groups/read-only/message"
}
```

## `GET /api/v0/tool-groups/:name`

Returns the configuration and proxy endpoints for a single group.

**Access:** admin only

<ParamField path="name" type="string" required>
  Name of the tool group.
</ParamField>

```bash cURL
curl http://localhost:8080/api/v0/tool-groups/read-only \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Response `200`**

The response combines the group configuration fields (`name`, `description`, `included_tools`, `included_servers`, `excluded_tools`) with the endpoint URLs (`streamable_http_endpoint`, `sse_endpoint`, `sse_message_endpoint`).

```json
{
  "name": "read-only",
  "description": "Read-only filesystem tools",
  "included_tools": ["filesystem__read_file", "filesystem__list_directory"],
  "included_servers": [],
  "excluded_tools": [],
  "streamable_http_endpoint": "http://localhost:8080/v0/groups/read-only/mcp",
  "sse_endpoint": "http://localhost:8080/v0/groups/read-only/sse",
  "sse_message_endpoint": "http://localhost:8080/v0/groups/read-only/message"
}
```

## `GET /api/v0/tool-groups/:name/effective-tools`

Resolves and returns the complete list of tool names that are actually active in the group after all inclusions and exclusions are applied. This is the ground truth for what an MCP client connecting to the group's proxy endpoint will see.

**Access:** admin only

<ParamField path="name" type="string" required>
  Name of the tool group.
</ParamField>

```bash cURL
curl http://localhost:8080/api/v0/tool-groups/all-weather/effective-tools \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Response `200`**

```json
{
  "tools": [
    "weather__get_forecast",
    "weather__get_alerts",
    "weather__get_current"
  ]
}
```

<ResponseField name="tools" type="string[]" required>
  Resolved list of tool names visible through this group's MCP proxy.
</ResponseField>

## `PUT /api/v0/tool-groups/:name`

Replaces the configuration of an existing tool group. All fields in the request body overwrite the current configuration — there is no partial update.

**Access:** admin only

<ParamField path="name" type="string" required>
  Name of the group to update.
</ParamField>

The request body accepts the same fields as `POST /api/v0/tool-groups`.

```bash cURL
curl -X PUT http://localhost:8080/api/v0/tool-groups/read-only \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "read-only",
    "description": "Updated read-only tools",
    "included_tools": [
      "filesystem__read_file",
      "filesystem__list_directory",
      "filesystem__get_file_info"
    ]
  }'
```

**Response `200`**

<ResponseField name="name" type="string" required>
  Name of the group that was updated.
</ResponseField>

<ResponseField name="old" type="object" required>
  Previous configuration of the group.
  <Expandable title="old properties">
    <ResponseField name="name" type="string">Previous name.</ResponseField>
    <ResponseField name="description" type="string">Previous description.</ResponseField>
    <ResponseField name="included_tools" type="string[]">Previous included tool names.</ResponseField>
    <ResponseField name="included_servers" type="string[]">Previous included server names.</ResponseField>
    <ResponseField name="excluded_tools" type="string[]">Previous excluded tool names.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="new" type="object" required>
  Updated (now live) configuration of the group. Same shape as `old`.
</ResponseField>

## `DELETE /api/v0/tool-groups/:name`

Deletes a tool group. MCP clients that were pointed at the group's proxy endpoint will receive `404` responses after deletion.

**Access:** admin only

<ParamField path="name" type="string" required>
  Name of the group to delete.
</ParamField>

```bash cURL
curl -X DELETE http://localhost:8080/api/v0/tool-groups/read-only \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Response `204`** — No content.

## MCP proxy endpoint for a group

```
ANY /v0/groups/:name/mcp
```

This is the streamable HTTP MCP endpoint for the named tool group. Point an MCP client at this URL to give it access to exactly the tools in the group, and no others. The URL is returned automatically when you create or fetch a group.

```bash Configure an MCP client
{
  "mcpServers": {
    "read-only-tools": {
      "url": "http://localhost:8080/v0/groups/read-only/mcp",
      "headers": {
        "Authorization": "Bearer CLIENT_ACCESS_TOKEN"
      }
    }
  }
}
```

<Tip>
  A single MCPJungle instance can serve multiple tool groups simultaneously. Create one group per client or per use-case, each with a different subset of tools, and hand out the corresponding `/v0/groups/:name/mcp` URL.
</Tip>

An SSE transport alternative is also available at `/v0/groups/:name/sse` (with the companion message endpoint at `/v0/groups/:name/message`), but the streamable HTTP endpoint is preferred for new integrations.
