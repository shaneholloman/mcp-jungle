Mcpjungle aggregates multiple MCP servers into a single MCP endpoint.

It acts as a gateway that your AI clients connect to, providing unified access to tools, prompts, and resources across all MCP servers.

## Use Cases
Mcpjungle is primarily built for:

- **Individuals** who want to use MCPs locally through one clean endpoint, no configuration hassle.
- **Teams** that want centralized MCP management, shared access to MCPs and Governance - hosted in their own cloud.

## Why Mcpjungle?

Without a gateway, MCP usage does not scale cleanly.

Every new MCP server means more duplicated client config, more scattered credentials, and less visibility into what tools are actually available.

Mcpjungle introduces a single control layer for that sprawl.

## Quickstart

<Steps>
  <Step title="Start the server">
    Pull the Docker Compose file and start Mcpjungle with a persistent Postgres database:

    ```bash
    curl -O https://raw.githubusercontent.com/mcpjungle/MCPJungle/refs/heads/main/docker-compose.yaml
    docker compose up -d
    ```
</Step>
</Steps>
