If you are using, evaluating, or contributing to mcpjungle, these are the best ways to reach the core team.

## GitHub Issues

Use [GitHub Issues](https://github.com/mcpjungle/MCPJungle/issues) when you want to:

- report a bug
- request a feature
- highlight a docs problem
- track a concrete piece of work

Issues are the best place for actionable, trackable items.

## GitHub Discussions

Use [GitHub Discussions](https://github.com/mcpjungle/MCPJungle/discussions) when you want to:

- ask broader product or usage questions
- discuss design ideas
- share feedback before opening a concrete issue
- talk about MCP workflows and deployment patterns

Discussions are the best place for open-ended conversation that is not yet a specific bug or task.

## Discord

Join the [mcpjungle Discord](https://discord.gg/CapV4Z3krk) if you want:

- faster back-and-forth with the team and community
- help while trying something out
- informal discussion around new ideas
- a place to stay close to what the project is doing

Discord is the best place for quick conversation. If something turns into a concrete bug or feature request, move it to GitHub Issues so it can be tracked properly.

## Contributing

If you want to contribute code, docs, or product feedback, start with whichever of the channels above best fits your question.

For concrete contribution workflow details, see:

- [Contributing](/developers/contributing)
- [Development guide](/developers/development)
